# Agent 设计

## 1. 资料解析 Agent

职责：把课程资料转化为结构化知识点。

输入：文件列表、课程名、考试日期、教师划重点。  
输出：章节、知识点、关键词、题型、易错点。

## 2. 考点分析 Agent

职责：分析往年卷中知识点出现情况。

核心字段：

- `examWeight`：考试重要程度
- `questionTypes`：题型分布
- `scoreRatio`：分值占比
- `frequency`：出现频率

## 3. 人类化讲解 Agent

职责：降低理解门槛。

固定讲解模板：

```text
这是什么：
为什么重要：
考试怎么考：
生活化类比：
例题演示：
常见错误：
```

## 4. 阶段性主动推进 Agent

职责：减少用户思考 Prompt 的成本。

推进节点：

- 讲完概念后：继续下一层吗？
- 讲完例子后：要不要做小测？
- 做错题后：要不要换一种讲法？
- 学完一节后：加入思维导图吗？
- 完成一章后：进入综合题吗？

## 5. 记忆方法 Agent

职责：针对知识类型给不同记忆方法。

| 类型 | 策略 |
|---|---|
| 概念类 | 口诀、类比、对比表 |
| 公式类 | 符号拆解、推导逻辑 |
| 流程类 | 步骤图、顺口溜 |
| 混淆类 | 对照卡片 |
| 简答题 | 关键词骨架 + 分点展开 |

## 6. 出题与批改 Agent

职责：根据当前知识点自动生成题目，并对答案给出反馈。

输出：

- 题目
- 参考答案
- 模拟分数
- 命中关键词
- 缺失关键词
- 失分原因
- 下一步建议
