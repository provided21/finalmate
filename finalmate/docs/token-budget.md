# Token 预算估算

FinalMate 的 Token 消耗主要来自：

1. 资料解析
2. 往年卷分析
3. 多轮讲解
4. 主动推进
5. 记忆方法生成
6. 出题与批改
7. 错题复盘

## 单门课估算

| 模块 | Token 估算 |
|---|---:|
| 资料解析 / OCR / 切分 | 190k |
| 往年卷分析 / 热力图 | 60k |
| 讲解 + 主动推进 | 320k |
| 记忆方法 + 思维导图 | 120k |
| 出题与批改 | 160k |
| 错题本 / 复盘 | 70k |
| 合计 | 920k |

实际深度复习周期可以按 150 万至 300 万 tokens 估算。

## 降本策略

- 课程资料解析后缓存结构化结果。
- 高频系统 Prompt 做缓存。
- 非实时任务走 Batch。
- 小题批改使用轻量模型。
- 热力图和思维导图优先用规则生成。
- 仅在学生追问时做深度讲解。
